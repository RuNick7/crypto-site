\section{Введение}

\subsection{Цель проекта}

«Секретный Архив» — статический образовательный веб-сайт, позволяющий пользователю
в интерактивном формате изучить три ключевых эпизода истории военной криптографии
XX века. Сайт построен без использования фреймворков и сборщиков — чистый HTML5,
CSS3 и нативный JavaScript с ES-модулями.

Основные образовательные цели:
\begin{itemize}[leftmargin=2em]
  \item объяснить математическую основу каждого шифра через пошаговые визуализации;
  \item предоставить работающие симуляторы для самостоятельного экспериментирования;
  \item завершить каждую операцию проверочным заданием с валидацией ответа;
  \item показать реальный исторический контекст и последствия взлома шифров.
\end{itemize}

\subsection{Принципы архитектуры}

\textbf{Разделение логики и представления.} Вся криптографическая математика
вынесена в отдельные ES-модули (\texttt{js/ciphers/}), которые не содержат
ни единой ссылки на DOM. HTML-страницы импортируют эти модули через
\texttt{<script type="module">} и используют функции для построения UI.

\textbf{Система допусков.} Прохождение каждой операции записывается в
\texttt{localStorage} (ключи \texttt{done\_vigenere}, \texttt{done\_enigma},
\texttt{done\_venona}). Навигационная панель на всех страницах читает эти
значения и показывает счётчик «ДОПУСК N/3».

\textbf{Дизайн-система.} Все цвета, отступы и шрифты определены как CSS-переменные
в \texttt{base.css}. Стиль «тёмный терминал»: фон \texttt{\#0a0a0f}, акценты
янтарного (\texttt{\#f5a623}) и матричного зелёного (\texttt{\#00ff41}).

\section{Операция VIGENÈRE (1914–1918)}

\subsection{Математическая основа}

Шифр Виженера является полиалфавитной подстановкой. Для ключа длины $m$ и
открытого текста $p_0, p_1, \ldots$ шифртекст вычисляется как:

\begin{equation}
  c_i = (p_i + k_{i \bmod m}) \bmod 26
\end{equation}

где $k_j$ — позиция $j$-го символа ключевого слова в латинском алфавите.
Обратное преобразование (расшифрование):

\begin{equation}
  p_i = (c_i - k_{i \bmod m} + 26) \bmod 26
\end{equation}

\subsection{Шаги обучения}

Операция реализована в виде пяти последовательных шагов:

\begin{enumerate}[leftmargin=2em]
  \item \textbf{Таблица Виженера} — интерактивная визуализация: пользователь вводит
        открытый текст и ключ, каждая пара (буква + сдвиг) отображается в ячейке
        с анимацией появления.

  \item \textbf{Шифратор/расшифровщик} — свободное шифрование с пошаговым разбором:
        для каждой буквы показаны исходная буква, буква ключа и результат.

  \item \textbf{Метод Казиски} — поиск повторяющихся триграмм в шифртексте.
        Расстояния между совпадениями кратны длине ключа; НОД даёт вероятную
        длину ключа.

  \item \textbf{Частотный анализ} — разбивка шифртекста на $m$ колонок (каждая
        является шифром Цезаря), визуальные гистограммы частот, автоматическое
        предположение сдвига по самой частой букве.

  \item \textbf{Задание-взлом} — фиксированное перехваченное сообщение с известным
        ответом (ключ \texttt{WAR}), валидация ввода пользователя.
\end{enumerate}

\subsection{Реализация криптографии}

\begin{lstlisting}[style=jsStyle, caption={Шифрование Виженера (vigenere.js)}]
function encrypt(plaintext, keyword) {
  const pt  = normalize(plaintext);
  const key = normalize(keyword);
  if (!key.length) return pt;
  return pt.split('').map((ch, i) => {
    const shift = ALPHA.indexOf(key[i % key.length]);
    return ALPHA[(ALPHA.indexOf(ch) + shift) % 26];
  }).join('');
}
\end{lstlisting}

\subsection{Задание с проверкой ответа}

Для финального шага определено фиксированное задание:

\begin{itemize}[leftmargin=2em]
  \item \textbf{Открытый текст:} \texttt{GERMANADVANCEONPARISALLUNITSMOVEFORWARD}
  \item \textbf{Ключ:} \texttt{WAR}
  \item \textbf{Шифртекст:} вычисляется при загрузке страницы функцией \texttt{encrypt}
\end{itemize}

При вводе неверного ключа пользователь получает:
\begin{enumerate}[leftmargin=2em]
  \item анимацию дрожания поля ввода (\texttt{.shake});
  \item красное сообщение с подсказкой (применить частотный анализ);
  \item кнопку «Узнать ответ с объяснением» — пошаговое раскрытие алгоритма:
        Казиски $\to$ длина ключа~3, частотный анализ трёх колонок $\to$ W, A, R.
\end{enumerate}

\section{Операция ENIGMA (1939–1945)}

\subsection{Принцип работы машины}

Энигма — электромеханическая полиалфавитная машина. При нажатии клавиши
электрический сигнал:

\begin{enumerate}[leftmargin=2em]
  \item проходит через штекерную панель (перестановка пар букв);
  \item последовательно через три ротора (вперёд);
  \item отражается рефлектором;
  \item проходит через три ротора в обратном направлении;
  \item снова через штекерную панель.
\end{enumerate}

\textbf{Ключевое свойство:} рефлектор делает шифрование симметричным, и буква
\emph{никогда} не шифруется сама в себя. Это стало роковой уязвимостью.

Общее число конфигураций Энигмы I:

\begin{equation}
  N = \underbrace{60}_{\text{выбор роторов}} \times
      \underbrace{17576}_{\text{позиции}} \times
      \underbrace{17576}_{\text{кольца}} \times
      \underbrace{\sim 10^{14}}_{\text{штекерная панель}}
  \approx 1.58 \times 10^{20}
\end{equation}

\subsection{Реализация EnigmaMachine}

Класс \texttt{EnigmaMachine} (197 строк) реализует полную симуляцию:
роторный шаг с двойным проскоком (\textit{double-stepping anomaly}),
прямое и обратное прохождение через роторы, рефлектор, штекерную панель.

\begin{lstlisting}[style=jsStyle, caption={Шаг роторов с двойным проскоком}]
step() {
  const notches = this.rotorDefs.map(r => r.notch);
  const pos = this.positions;
  const midAtNotch   = ALPHA[pos[1]] === notches[1];
  const rightAtNotch = ALPHA[pos[2]] === notches[2];
  if (midAtNotch) {
    pos[0] = (pos[0] + 1) % 26;
    pos[1] = (pos[1] + 1) % 26;
  }
  if (rightAtNotch) pos[2] = (pos[2] + 1) % 26;
  else if (midAtNotch) {} // already stepped
  else pos[2] = (pos[2] + 1) % 26;
}
\end{lstlisting}

\subsection{Перехваченное немецкое сообщение}

Для учебного задания выбрана реальная военно-морская сводка погоды —
исторически наиболее важный источник крибов для Блетчли Парк:

  \begin{tabular}{ll}

    \textbf{Параметр}   & \textbf{Значение} \\
    \midrule
    Открытый текст      & \texttt{WETTERBERICHTNORDSEEGEBIETBEWOELKTLEICHTESWINDE} \\
    Роторы              & III -- II -- I \\

    Штекерная панель    & нет \\

Именно такие сводки Кригсмарине отправлял ежедневно в 06:00. Слово

открытого текста, с которого начиналась атака Бомбы каждое утро.

\subsection{Алгоритм Бомбы Тьюринга}

Исправленный алгоритм \texttt{bombeSearch} перебирает все $26^3 = 17\,576$
стартовых позиций роторов и для каждой проверяет, совпадает ли расшифрованный
текст с крибом:

\begin{lstlisting}[style=jsStyle, caption={Исправленная логика bombeSearch}]
// Для каждой позиции [r, m, l]:
let valid = true;
for (let i = 0; i < crib.length && i < ct.length; i++) {
  const result = machine.encodeLetter(ct[i]); // decode (Enigma симметрична)
  if (!result || result.output !== crib[i]) {
    valid = false;
    break;
  }
}
if (valid) results.push({ positions: [r,m,l], posStr: ... });
\end{lstlisting}

\textbf{Результат:} для нашего перехваченного сообщения и криба \texttt{WETTER}
алгоритм находит ровно \textbf{1 совпадение} — позицию \texttt{AAA}.

\subsection{Шаги обучения}

\begin{enumerate}[leftmargin=2em]
  \item \textbf{Схема сигнала} — анимированная SVG-диаграмма прохождения сигнала
        через компоненты машины с кнопкой «⚡ ПРОСЛЕДИТЬ СИГНАЛ».
  \item \textbf{Симулятор} — полноценная клавиатура, лампочки, дисплей роторов,
        поддержка произвольных настроек.
  \item \textbf{Криб-драггинг} — таблица возможных/невозможных позиций криба.
  \item \textbf{Бомба Тьюринга} — анимированный перебор с барабанами, прогресс-баром,

\end{enumerate}

\section{Операция VENONA (1943–1980)}

\subsection{Одноразовый блокнот (OTP)}

Одноразовый блокнот — единственная криптосистема с \textit{доказуемой совершенной
секретностью} (теорема Шеннона, 1949). Для ключа $k$ длины $n$:

\begin{equation}
  c_i = (p_i + k_i) \bmod 26
\end{equation}

Без знания $k$ любой открытый текст длины $n$ является математически
равновероятным объяснением шифртекста.

\subsection{Роковая ошибка НКВД}

В 1942–43 гг. из-за нехватки ключевых блокнотов часть ключей была использована
дважды. Если два сообщения $P_1$, $P_2$ зашифрованы одним ключом $K$:

\begin{align}
  C_1 &= P_1 \oplus K \\
  C_2 &= P_2 \oplus K \\
  C_1 \oplus C_2 &= P_1 \oplus P_2
\end{align}

Ключ исчезает! XOR двух осмысленных текстов содержит статистические
паттерны языка, которые восстанавливаются через криб-драггинг.

  \item переводит пользователя на шаг~0 текущей операции.
\end{itemize}

\section{Образовательные результаты}

\subsection{Покрытые темы}

\begin{enumerate}[leftmargin=2em]
  \item \textbf{Полиалфавитное шифрование} — принцип периодического ключа,
        таблица Виженера, слабость к частотному анализу.
  \item \textbf{Метод Казиски} — обнаружение длины ключа по повторяющимся
        триграммам, вычисление НОД расстояний.
  \item \textbf{Индекс совпадений} — статистическое различение случайного
        шифртекста (ОТП) от периодического (Виженер).
  \item \textbf{Электромеханические шифраторы} — принцип роторной машины,
        двойное проскакивание, рефлектор и его уязвимость.
  \item \textbf{Криб-драггинг} — использование известных фрагментов открытого
        текста для исключения позиций.
  \item \textbf{Атака перебором с ограничениями} — принцип Бомбы Тьюринга,
        разница между перебором и логическим исключением.
  \item \textbf{Совершенная секретность OTP} — доказательство Шеннона,
        равномерность частот в шифртексте.
  \item \textbf{Атака глубины} — повторное использование OTP-ключа,
        $C_1 \oplus C_2 = P_1 \oplus P_2$.
\end{enumerate}

\subsection{Геймификация}

Система «допусков» (0/3 → 1/3 → 2/3 → 3/3) мотивирует пройти все три операции.
Прогресс-бар в нижней части каждой страницы визуализирует общее продвижение.

\section{Заключение}

Проект реализует образовательную платформу по истории криптографии, охватывающую
три ключевых эпохи: Первую мировую (шифр Виженера), Вторую мировую (машина Энигма)

        перезапуска операции;
  \item реальное немецкое сообщение (сводка погоды Кригсмарине) с переводом
        и историческим комментарием.
\end{itemize}
